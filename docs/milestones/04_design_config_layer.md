# Milestone 4 — Design the Configuration Layer

Inside `config/`, create:
- `project_config.yml`
- `paths_config.yml`
- `tables_config.yml`

## Why split config?
Because one giant config file becomes hard to navigate.
Split config helps separate:
- project identity
- paths and storage design
- table naming

## Day 1 mindset
You are not filling every value perfectly yet.
You are defining the configuration architecture that the rest of the week will use.
