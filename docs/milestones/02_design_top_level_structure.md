# Milestone 2 — Design the Top-Level Structure

Create the top-level folders in your own repository:

- `config/`
- `docs/`
- `sample_data/`
- `sql/`
- `src/`
- `notebooks/`
- `tests/`
- `.github/workflows/`

## Why this matters
A flat project becomes messy very quickly.
A structured project tells the reader where to look for:
- settings
- explanation
- data
- SQL
- code
- notebook logic
- testing
- automation checks

## Engineering idea
This is where you begin to show engineering sense, not only coding ability.
